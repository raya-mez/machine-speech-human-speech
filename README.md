# Machine Speech, Human Speech

## Project Overview
As large language models (LLMs) increasingly populate everyday writing, certain words and stylistic habits have come to be perceived as telltale signs of machine authorship, reportedly prompting some writers to avoid them. This project investigates this oppositional pathway of LLM-driven language change, in which human writers adapt their language to not resemble LLM output but to be read as unmistakably human. Because LLMs themselves learn from human language which may include these oppositional changes, we hypothesized that successful differentiation strategies would not remain exclusively human for long, making this an iterative process of adaptation and counter-adaptation. To test this, we designed an online experiment repurposing the chain-generation structure of the Iterated Learning paradigm. Participants iteratively edited a news article to make it read more convincingly as human-written and unlike a simulated, state-of-the-art LLM that had itself incorporated the editing strategies of previous participants. We assessed (1) the sustainability of this process, based on participants' own judgments of whether their edits improved the text's human-authorship signal; (2) whether the strategies underlying participants' edits changed systematically across generations; and (3) whether such changes were reflected in the lexical diversity, syntactic complexity, and orthographic and grammatical correctness of the texts themselves. No chain terminated early due to successive participants judging their editing as unsuccessful, indicating that participants could sustain the differentiation process across repeated rounds. Editing rationales showed no consistent linear change across generations, though exploratory analyses suggested a possible non-monotonic, reaction-and-counterreaction pattern along some dimensions. At the level of the texts, lexical diversity declined steadily across generations, while syntactic complexity showed no comparable systematic change and error rate only increased slightly in early generations. These findings provide some of the first controlled experimental evidence for a sustained, oppositional process of identity signaling against LLMs, while also suggesting that the absence of any mechanism for writers to coordinate may direct this process toward simplification rather than the lexical innovation more typical of language used for identity differentiation.

## Repository Structure
- `data/` 
	- `raw/` contains anonymized survey responses.
	- `processed/` contains processed versions for the analyses.
	- `dynamic` contains a file storing whether chains terminate at each generation (used during data collection to determine which chains to continue).
- `analysis/` contains the analysis notebooks and output tables for the research questions.
	- `explore_and_process_data.ipynb` contains initial exploratory analyses and processing.
	- `RQ1_sustainability.ipynb` contains analyses related to RQ1, specifically inspections of the responses of participant who judged their editing as unsuccessful.
	- `RQ2_explanations_processing.ipynb` further processes the response data to prepare the confirmatory analyses related to RQ2 about editing explanations, which are conducted in `RQ2_explanations_stats.ipynb`.
	- `RQ3_features_processing.ipynb` similarly prepares the confirmatory analyses related to RQ3 about text features conducted in `RQ3_features_stats.ipynb`.
	- `p_values_features.csv` and `p_values_explanations.csv` store p-values obtained from the confirmatory analyses conducted in `RQ2_explanations_stats.ipynb` and `RQ3_features_stats.ipynb`, respectively.
	- `analysis_significance.ipynb` contains the correction of the p-values for multiple comparisons across all confirmatory analyses.
- `experiment/` contains the survey-generation scripts and Qualtrics reference material.
	- `materials/` contains the scripts used to generate the initial survey materials (the news articles given for editing at Generation 1), the materials themselves, and a JSON file containing the materials and additional data used to generate the survey for each generation.
	- `survey_files/` stores the Qualtrics survey files for each generation, in addition to one template file from which the former are built.
	- `build_surveys.py` contains the script to build a survey file for a given generation based on the materials and data in the corresponding JSON.
	- `create_next_gen_survey_input_data.py` builds the JSON containing the materials for the next generation.
	- `QSF_format_info.md` documents the structure of the Qualtrics survey files.
- `experiment_utils/` contains shared helper functions.
